O CSS tem uma espécie de hierarquia aplicada de acordo com o que é colocado no HTML, onde uma id é o topo, uma classe é o meio e uma tag é a menor de todas, quando referenciamos uma tag com um caracteristica e referenciamos essa mesma tag com uma outra tag na qual ela está dentro e com uma caracteristica oposta a primeira feita, essa segunda será priorizada, pois ela é quem tem a força de duas tag e ela é acima de uma tag normal, mas continua abaixo de uma classe. Para uma forma mais simples de analisar, segue um exemplo matematico, onde ID é 100, calsse é 50 e tag é 1, quando temos duas tags elas meio que somam esse valor assim sendo 2 e sendo assim maior que uma unica tag. Apenas uma coisa é maior que o um ID que é a configuração CSS Inline e isso se deve por estar diretamente no elemento, no exemplo matematico ela seria igual a 1000 e nada consegue sobrepor essa prioridade no CSS

Vale ressaltar que o CSS é um efeito em cascata então uma caracteristica sempre será ofuscado pela mesma caracterisca que estiver abaixo dela, por isso, há formas de contornar isso, colocando ambas as caracteriscas na mesma linha

Pseudo elementos são elementos que criamos via CSS e não existem no HTML,elas vão na frente de uma seleção como o hover etc. por exemplo:first-letter(pra selecionar a primeira letra de um texto), first-line(para a primeira linha), after(aplica um caractere que não é lido como um HTML, mas sim CSS), before(funciona da mesma forma que o After, a diferença é que before aplica antes do texto HTML e o After aplica depois do texto, outra coisa que é importante ressaltar é que amabas funcionam apenas com a caracteristica visual Content), not(esse aplica uma excesão no que vai receber um CSS deixando a excesão entre parenteses ao lado do not)

Caracteristicas visuais

  • Font-size = altera o tamanho da fonte medida em pixels
  • Text-Align = se refere ao alinhamento do paragrafo podendo centralizar ele na pagina
  • Background = aplica uma cor hexadecimal como fundo de um texto
  • color = muda a cor da letra
  • src = busca um arquivo externo
  • height = mexe na altura do elemento por px ou %
  • width = mexe na largura por px ou %
  • padding = cria um espaçamento interno em px ou em %(aumenta o conteudo)
  • Margin = Cria um espaçamento externo em px ou em %(aumenta o espaçamento)
  • font-style = altera o estilo da fonte, podendo colocar negrito ou em italico
  • display = altera a visualização entre inline(fecha a linha idependente do espaço que ela ocupa, block(fecha o conteudo no espaço dela) e flex(transforma o elemento em uma Flex-container, que permite todos os elementos descendentes diretos possam ser remanejados) grid(transforma o elemento em um Grid-container que tem quase a mesma utilidade do Flex, porem consegue aplicar coisas que um flex-container não conseguiria e vice versa
  • Text-transform = transforma o texto mudando suas caracteristicas como o visual para parecer maiusculo com Uppercase
  • Font-weight = muda o peso da fonte, podendo colocar a opção negrito(bold) como exemplo
  • Text-decoration = muda a propriedade de exibição do texto(none para tirar as decorações/underline para colocar sublinhado)
  • Position = altera a posição entre static(posição natural dele), absolute(posição absoluta em relação a pagina inteira) e relative(posição relativa ao natural, podendo mexer na posição), utilizando depois as caracteristica de localizão(top,botton,left,right)
  • auto = é um resultado de localização(% ou px) usado para a propria pagina calcular algum tamanho, no exemplo foi pego os valores laterais, somados e divididos por 2 para centralizar o cabeçalho
  • vertical-align = serve para orientar o alinhamento do texto que muitas vezes segue o alinhamento do texto de baixo e caso o alinhamento deva seguir o texto de cima é usado essa caracteristica junto da orientação que no exemplo da aula foi top
  • Box-sizing = altera a forma vista em que os elementos são colocados, alterando a soma de porcentagem e pixels para que um não afete o outro usando o elemento border-box
  • border-color = dá uma cor à borda
  • border-width = dá uma grossura à borda
  • border-style = dá um estilo à borda, dentre elas temos o solid que é uma barra solida sem muito estilo, temos tbm o Dashed que é o tracejado, o dotted que é pontilhado
  • border-radius(com escolha de direção para ecolher onde arredondar) = essa caracteristica visual, aplica o arredondamento da borda aplicando com uma base matematica no conceito de raio de um circulo e o pondo em pratica com px
  • transition = essa caracteristica define uma transição de efeitos visuais, colocando nela o tempo que a transição dura e onde ela vai se aplicar, colocando o nome do efeito que sofrera a mudança do hover, para fazer uma transição de todos os elementos do item, é usado a resposta all
  • cursor = altera a visualização do cursor, pointer para a suposta mãozinha que é identificavel como item clicavel
  • transform = serve para mudarmos o tamanho(scale() e dentro do parenteses é colocado uma multiplicação de quanto vai ser aumentado proporcionalmente todos os itens) ou o comportamento do elemento(rotate(deg) dentro dos parenteses é colocado a numeração e medida em graus que no inglês é colocado com "Deg" na abreviação)
  • float = essa caracteristica facilita no quesito display e position, uma forma de mesclar um texto e uma imagem na mesma linha, mas sem deixar um espaço em branco que o display deixa ou um trabalho enorme para ter um resultado aceitavel com position, com o float, escolhemos o lado em que a imagem vai ficar e assim fazer com que o conteudo recarregue ao redor do float
  • Clear = essa caracteristica se trata de um oposto do float, pois o float afeta toda a estrutura da pagina e com o clear, podemos delimitar uma area em que o float afeta
  • line-height = altera a altura da linha de texto, criando esse espaçamento
  • linear-gradient = essa caracteristica é aplicada ao lado do background, como resposta, sua estrutura monta um efeito gradiente na selecionando entre parenteses as cores escolhidas para o efeito e separando elas por virgulas, funciona tanto com a nomeclatura da cor quanto com o codigo hexadecimal, colocando porcentagens ao lado das cores, definimos o inicio de cada cor, a ordem posta define a ordem vista; também é possivel alterar a inclinação do gradiente usando no inicio dos parenteses o angulo em deg
  • radial-gradient = funciona da mesma forma que o linear gradient, só que invés de linhas, é feito um gradient redondo
  • Content = aplica entre as aspas um caractere em CSS, necessario o uso dos pseudos elementos After e Before
  • Opacity = essa caracterisca altera a opacidade de um objeto, indo de valores de 1(100%) a 0(0%) para usar valores entre eles, é usado numeros quebrados como por exemplo 0.3 para 30% e assim vai, a opacity também pode ser aplicada em cores, fazendo uma transferencia de hexadeciamal, ou nomeclatura, para rgba() e dentro do parenteses vai a numeração das cores representadas, separadas por virgula, e terá um quarto elemento, conhecido como alpha, mas em uma tradução mais facil de entender, é o valor da opacidade que funciona da mesma forma de 1 a 0
  • Box-shadow = coloca uma sombra no elemento onde os valores colocados após ele são numericos e é composto por 5 valores, o primeiro é o eixo X, o segundo é o eixo Y, e antes que haja alguma pergunta, o terceiro não é um eixo Z já que não temos acesso a um 3D, mas o terceiro se trata do blur da sombra, o quarto é a intensidade da borda que a partir do eixo dela, ela terá um tamanho maior, imagine como se aproximasse uma fonte de luz de um objeto onde a sombra aumenta gradativamente, e por fim o ultimo valor é uma cor que será aplicada na sombra; mesmo envolvendo tantos valores, é possivel aplicar varias sombras, as separando na mesma linha por virgulas e recriando os mesmos 5 valores, podendo aplicar até mesmo opacidade usando o rgba()
  • Inset = sub-caracteristica de Box-shadow, ela é responsavel por fazer a sombra ser projetada internamente do objeto onde os unicos valores usaveis nela são a intensidade e a cor pois esse tipo de sombra não é deslocavel então no lugar dos eixos X e Y colocamos 0 para chegarmos a intensidade
  • text-shadow = funciona da mesma forma que o box shadow, só que aplica em textos
  • box-sizing = ajeita o tamanho de elementos para encaixar sem restrições ao tamanho necessario, caso o elemento seja maior que o pedido e não consiga ser colocado em tal redimensionamento, o Box-sizing criara um overflow onde o conteudo será ajeitado para aquela dimensão
  • Z-index = essa caracteristica é o eixo Z, aquele que mexe com a profundidade de um objeto, com ele podemos puxar objetos a frente de outros utilizando valores numericos base(1,2,3,4...), por padrão todos são definidos como auto, ou seja, todos se agrupam de acordo com as linhas e colocados como 0, caso um tenha o valor maior que esse, logo terá maior destaque,sobrepondo os que estavam a sua frente, dá para fazer um efeito oposto utilizando de numeros negativos, assim os colocando atrás dos auto
  • Justify-content = em uma flex-container, ele é o responsavel em ajeitar horizontalmente os itens usando comandos como flex-start, flex-end, center, sapace-around, space-between e space-evenly; esse tipo de comando apenas realoca itens dentro de um espaço, se o item for maior que o Flex-container, pode não funcionar corretamente
  • Aling-items = se justify é responsavel por realocar horizontalmente os elementos de uma Flex-container, o aling-itens realoca verticalmente, usando dos comands (flex-start, flex-end, center, stretch) vale ressaltar a diferença entre Stretch e center, amabos centralizão os botões mas, enquanto o center centraliza o botão, stretch estica ele para que preencha todo o vertical
  • Flex-wrap = há casos em que um Flex-container tenha muito conteudo, mostrando incapacidade de quebrar a linha automaticamente, o Flex-wrap é o responsavel por essa quebra para alinhar na tela sem cria um scroll lateral na pagina, para isso essa caracteristica tem duas resposta, o nowrap que é a config. padrão dele e o wrap que libera a permisão de quebra de linha
  • Flex-direction = essa caracteristica aplicada em Flex-containers, serve para guiar a direção em que o Flex container vai preencher usando as respostas Row(linha) ou Column(coluna), dependendo do resultado isso pode influencia no Justify-content e Aling-items, os invertento
  • Aling-self/Justify-self = essas duas caracteristicas são aplicadas quando queremos alinhar separadamente um elemento do flex-container, e para isso, ele não é colocado diretamente no CSS, mas sim no HTML com o style, dentro da tag que deseja separar das demais
  • Grid-template-columns = sendo utilizado apenas em Grid-containers ele consegue ajeitar um grid-container na questão de colunas, quantas linhas queremos e o tamanho delas, junto dessa caracteristica, podemos utilizar como resposta uma nova unidade, chamada fr ou fração, essa unidade serve para não utilizarmos numeros exatos como 3 colunas de mesmo tamanho e invés de usar a porcentagem 33.33333... nunca iriamos terminar pois é um resultado constante e infinito, nesse caso utilizamos a fração, colocando 3 vezes o 1fr, a pagina logo entendera que se trata de uma fração de 1/3(um terço)
  • Grid-template-rows = funciona da mesma forma que o grid-template-rows
  • Overflow = especifica quando o conteúdo de um elemento de nivel de bloco deve ser cortado(hidden), exibido com barras de rolagem ou se transborda do elemento.
  • Grid-colums/rows = ele dá uma caracteristica especifica do grid para a coluna/linha que receber essa caracteristica:Span(mescla as celulas do grid espeficando a quantidade de colunas/linhas que deve mesclar) n/n(esse define a possição do item, definindo seu ponto inicial e ponto final, onde o primeiro numero é o ponto inicial e o segundo numero é o ponto final)
  • Gap = cria um espaçamento em um grid, muito semalhante ao margin
  • Background-image = puxa uma imagem utilizando o url() guiando da mesma forma que um src
  • Background-size = define um tamanho da imagem
  • Background-repeat = define se uma imagem irá se repetir ao ser colocada
  • Background-position = define a posição da imagem
  • Grid-template-area/grid-area = essa caracteristica ajuda a reposicionar e ajustar um grid de forma mais facil de executar, o grid-template-area cologa um nome pra cada espaço do grid, e o grid-area dá esses nomes aos elementos assim os posicionando nos espaços de mesma nomeclatura
  • Flex-Grow = Responsavel por aumentar proporcionalmente o tamanho de um elemento utilizando como se fosse uma conta de vezes onde a resposta dele será por quantas vezes o elemento deve crescer(1=vezes 2, 2=vezes 3 e assim por diante) essa caracteristica deve ser aplicado aos filhos de um flex-container invés do proprio, uma coisa que deve ser levada em conta é o fato de que gap, border e padding, influenciam no flex-grow
  • list-style-type:altera o elemento before de uma lista
